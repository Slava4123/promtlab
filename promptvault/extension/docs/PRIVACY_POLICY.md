# Политика конфиденциальности расширения «ПромтЛаб»

**Последнее обновление:** 2026-05-12
**Версия расширения:** 1.0.0
**Опубликовать на:** https://promtlabs.ru/legal/extension-privacy

---

## 1. Общие сведения

Браузерное расширение «ПромтЛаб» (далее — «Расширение») является клиентом сервиса ПромтЛаб (promtlabs.ru), принадлежащего ИП Ковальчук В. (далее — «Оператор»). Расширение позволяет пользователю работать со своей библиотекой AI-промптов и вставлять их в окна чатов поддерживаемых LLM-сервисов.

Настоящая Политика описывает, какие данные собирает расширение, как они используются и как пользователь может ими управлять. Политика разработана в соответствии с требованиями Федерального закона «О персональных данных» №152-ФЗ и **Chrome Web Store User Data Policy / Mozilla Add-on Policies**.

## 2. Какие данные собираются

### 2.1. Данные, передаваемые на сервер

При работе с Расширением на серверы Оператора по адресу `promtlabs.ru` передаются:

- **API-ключ** (`pvlt_*`) или **JWT access token** — для аутентификации пользователя;
- **Содержимое промптов** — при создании/редактировании промптов через интерфейс расширения;
- **События использования** — факты вставки промптов (без содержимого вставленного текста), для подсчёта статистики и лимитов;
- **Email и пароль** — только при использовании email/password логина (передаются через HTTPS, пароль никогда не сохраняется локально);
- **Часовой пояс браузера** (заголовок `X-Timezone`) — для корректного отображения дат.

### 2.2. Данные, хранящиеся локально в браузере

В `chrome.storage.local` сохраняются:

- API-ключ (`pvlt_*`) — для последующих запросов;
- Тема оформления (light / dark / system);
- Базовый URL сервера (по умолчанию `https://promtlabs.ru`);
- Идентификатор активной команды и коллекции;
- 20 последних использованных промптов (offline-кэш);
- Сохранённые значения переменных для шаблонов `{{var}}` (per-prompt);
- Флаги UX (просмотр онбординга, последний показ changelog);
- 5-минутный кэш списка промптов.

### 2.3. Данные с веб-страниц AI-сервисов

Расширение работает на следующих сайтах: `chatgpt.com`, `claude.ai`, `gemini.google.com`, `www.perplexity.ai`, `alice.yandex.ru`, `ya.ru`, `yandex.ru/alice`, `giga.chat`, `developers.sber.ru`, `chat.deepseek.com`, `chat.mistral.ai`, `le-chat.mistral.ai`, `chat.qwen.ai`.

На этих сайтах расширение:

- **Записывает** текст промпта в поле ввода (textarea / contenteditable) — только по явному действию пользователя (клик «Вставить»);
- **Читает** содержимое последнего сообщения AI — только при явном действии «Захватить ответ» в режиме цепочек;
- **НЕ собирает** историю диалогов, метаданные сессий, файлы или иную информацию без явного действия пользователя;
- **НЕ передаёт** содержимое AI-чатов или вводимых пользователем сообщений на серверы Оператора.

### 2.4. Технические данные

- **Версия расширения** (`X-Client` заголовок) — для совместимости и отладки;
- **HTTP статусы и ошибки** — для подсчёта частоты ошибок интеграции с LLM-сайтами (через self-hosted GlitchTip).

## 3. Telemetry и ошибки (Sentry / GlitchTip)

Расширение интегрировано с **самостоятельно-размещённым** GlitchTip (Sentry-совместимый API) по адресу `glitchtip.promtlabs.ru`. Данные не передаются третьим лицам.

**Что отправляется при ошибке:**
- Тип ошибки и стек-трейс;
- Имя браузерного хоста (например, `chatgpt.com`) — для диагностики проблем с селекторами;
- Версия расширения;
- ID пользователя (числовой, без email).

**Что НЕ отправляется:**
- Содержимое промптов;
- Значения переменных шаблонов;
- API-ключи и токены;
- Email, пароли, имена.

Данные с чувствительными ключами (`key`, `token`, `password`, `secret`) автоматически замещаются на `[REDACTED]` перед отправкой. Содержимое полей `content` и `text` заменяется на маркер `[N chars]` без самого текста.

**Отключение телеметрии:** Telemetry активируется только если установлен параметр `WXT_SENTRY_DSN` на этапе сборки расширения. Стандартные сборки в Chrome Web Store и Firefox Add-ons могут иметь телеметрию включённой — отключение через настройку `WXT_SENTRY_DSN=` (пустое значение).

## 4. Передача данных третьим лицам

Оператор **НЕ передаёт** персональные данные третьим лицам, за исключением следующих случаев:

- **OAuth-провайдеры** (Google, GitHub, Yandex) — при использовании входа через OAuth расширение перенаправляет пользователя на сайт провайдера. Сами провайдеры обрабатывают данные согласно их политикам.
- **T-Bank** — при оплате подписки (производится на сайте promtlabs.ru, не в расширении).
- **Правоохранительные органы** — только по официальному запросу в соответствии с законом.

## 5. Хранение и удаление данных

- **Локальные данные** (chrome.storage.local) удаляются автоматически при удалении расширения из браузера.
- **Серверные данные** (промпты, коллекции, аккаунт) удаляются по запросу пользователя через настройки веб-приложения (Настройки → Профиль → Удалить аккаунт).
- **Срок хранения** удалённых данных в корзине — 30 дней с возможностью восстановления.

## 6. Права пользователя

В соответствии с законом 152-ФЗ пользователь вправе:

- Получить информацию о своих обрабатываемых данных;
- Потребовать уточнения, блокировки или удаления данных;
- Отозвать согласие на обработку.

Для реализации прав направьте запрос на email: **support@promtlabs.ru** или используйте форму обратной связи в расширении.

## 7. Безопасность

- Все запросы между расширением и сервером передаются по HTTPS (TLS 1.2+).
- API-ключи хранятся локально в `chrome.storage.local` и никогда не передаются на сторонние ресурсы.
- Срок действия JWT access token — 15 минут (не используется в расширении, вместо него используется API-ключ).
- Расширение НЕ имеет permission `<all_urls>` — доступ только к 13 явно перечисленным доменам.

## 8. Запрашиваемые разрешения

| Permission | Назначение |
|------------|------------|
| `sidePanel` | Открытие боковой панели расширения (Chrome) |
| `storage` | Хранение API-ключа, темы, локального кэша |
| `activeTab` | Определение активной AI-вкладки (host detection) |
| `scripting` | Re-inject content-scripts при обновлении расширения |
| `contextMenus` | Меню «Сохранить выделение как промпт» |
| `host_permissions` (13 доменов) | Insert/capture только на perpetuated AI-сайтах + связь с promtlabs.ru |
