# PromptVault — Code Review (2026-05-07, v2)

> Полный архитектурный и качественный аудит, второй прогон. Stack: Go 1.25.7 + Chi v5 + GORM v2 + PostgreSQL 18 (backend), React 19.2 + Vite 8 (Rolldown) + TanStack Query v5 + Zustand v5 (frontend). Self-hosted SaaS на VPS в РФ, реальные платежи (T-Bank), MCP server v1.5.0, 61 миграция, 26 backend usecase-пакетов, 27 HTTP-пакетов, 74 frontend pages.

> **Этот отчёт перезаписывает `REVIEW_2026-05-07.md` — предыдущий прогон сохранён в `docs/archive/REVIEW_2026-05-07_v1.md` для diff'а.** Главное расхождение с v1: `lucide-react ^1.7.0` подтверждён как **легитимный** пакет (npm latest 1.14.0) — старый CR-16 «squat» был false-positive и здесь не повторяется. Зато v2 ловит **новый блокер**: `npm run build` падает на 15 TS2322-ошибках в `chains/*` страницах из-за рассогласования между Button-компонентом и `asChild`-prop'ом → production build не собирается прямо сейчас.

**Топ-7 must-fix перед следующим релизом:**

1. **CR-1 [DEPLOY-BLOCKER]** — `npm run build` падает с 15 TS2322 errors в chains-страницах. Production frontend deploy будет сломан. Либо вернуть `asChild` в `ButtonProps`, либо переписать `<Button asChild><Link/></Button>` патерн в chains/*.tsx на прямые `<Link/>` или `<a/>` стили. **30-45 минут.**
2. **CR-2 [SECURITY]** — `subscription.go:189-190` и `:413-414` используют `gorm.Save(&User{ID, PlanID:"free"})` → mass-overwrite. Один REFUNDED webhook стирает email/password_hash/role у юзера; админ теряет роль (privilege downgrade). CVSS 9.1. **5 минут на patch.**
3. **CR-3 [SECURITY]** — `delivery/http/auth/handler.go:146`: `if user.IsAdmin() && h.adminauth != nil` — DI-bug или refactor → admin login без TOTP, без warning-лога. CVSS 8.1. **5 минут.**
4. **CR-4 [SECURITY]** — OAuth callback неявно линкует verified-email провайдер к **неверифицированному локальному** аккаунту (auth.go:104, oauth.go:251-263) → account-takeover при условии что атакующий зарегил email раньше + жертва логинится через OAuth. CVSS ~6 (требует knowledge of victim email + race). Нет в v1 REVIEW.
5. **CR-5 [SECURITY]** — `PAYMENT_WEBHOOK_ALLOWED_IPS` defaults to `[]` → `ipallowlist.go:25` пропускает всё. Если переменная не выставлена в prod `.env`, единственная защита — SHA-256 подпись, и при leak Password атакующий может слать поддельные webhooks с любого IP. CVSS 7.5.
6. **CR-6 [PERF/RELIABILITY]** — 8 background loops + 3 fire-and-forget горутины не имеют `defer recover()`. Любая паника в template-render, JSON-corrupt step_outputs или nil-deref убивает loop **навсегда** (до restart). T-Bank renewals прерываются → revenue loss; trash не чистится; insights не считаются.
7. **CR-7 [TESTS]** — `usecases/chain/` (1191 LOC Phase 16 dark-launch): **0 unit-тестов**. Cycle detection / RBAC viewer-vs-editor / fork tier-gate / snapshot semantics — без покрытия. **`CHAINS_ENABLED=true` rollout в prod заблокирован** до закрытия этого finding'а (CLAUDE.md явно говорит «task #17 — тех.долг»).

  import (
      "log/slog"
      "runtime/debug"

      "github.com/prometheus/client_golang/prometheus"

  var loopPanics = prometheus.NewCounterVec(prometheus.CounterOpts{
      Name: "promptvault_loop_panics_total",
      Help: "Background loop panics that were recovered",

  func init() { prometheus.MustRegister(loopPanics) }

  func RunWithRecover(name string, fn func()) {
      defer func() {
          if r := recover(); r != nil {

### MJ-5 [SECURITY] — Refresh token replay revoke без log

      if err := s.tokens.RevokeChain(ctx, oldToken.ID); err != nil {
          slog.Error("oauth.refresh.revoke_chain_failed",

## Architectural Observations (long-term)

1. **Composition root (`app/app.go`)** сделан грамотно. Все зависимости явно собираются, late-binding через `Set*` методы (SetActivity, SetEmail, SetTierChangeNotifier, SetFeedbackRepository, SetLogoRepo). Pattern «optional setter» правильно используется для разрыва циклических зависимостей. Composition-root правильно владеет `*gorm.DB`, ниже его никто не видит — за исключением CR-2 (user_repo.Save), MJ-3 (gorm.ErrRecordNotFound leak), MJ-2 (testcleanup намеренно).

2. **Models — чистый пакет без циркулярок**. Импортируют только `time` и `gorm`. 27 файлов, никаких circular deps. Issue: типизация enum'ов не доведена до конца (см. type-1 .. type-12).

3. **Repository интерфейсы (`interface/repository/`)** — чистые. 27 файлов, gorm-импорта нет. `errors.go` правильно вводит `repo.ErrNotFound`, `repo.ErrConflict`. Из 23 реализаций корректно мапят 21; 2 файла утечкают gorm (см. MJ-3).

4. **Local interface pattern в `delivery/http/admin/handler.go:38-69`** (`AdminService`, `TOTPVerifier`, `AuditReader`, `HealthCounter`) — отличный пример Go idiomatic dependency-inversion. Этот паттерн стоит распространить на остальные handlers.

5. **Late-binding через setters** — компромисс для разрыва circular DI. Альтернатива — Functional Options Pattern (`NewService(repo, ..., WithActivity(svc), WithEmail(email))`).

6. **Usecase auth/auth.go (664 строк, 26 методов)** — пограничный кандидат на разбиение. Можно вынести `password.go` (Set/Forgot/Reset/Change).

7. **CRON_FAST_DEV паттерн в `app.go:233-243`** — элегантное решение для QA.

8. **Use of `panic` в `app.go:177, 408, 417` для fail-fast при загрузке embedded catalog'ов** — правильный подход.

9. **CLAUDE.md**: 30% backend-фич устарели (см. секцию «Расхождения с CLAUDE.md»). Ostensibly v1 REVIEW уже flag'нул это, но drift всё ещё present — fix не дошёл.
