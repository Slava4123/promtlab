> Дата: 2026-04-10

## Часть 1: Фичи с AI (требуют LLM-расходов)

### A1. AI Enhance / Rewrite / Analyze (уже реализовано)

**Что есть:** Улучшение, рерайт и анализ промптов через OpenRouter (Claude Sonnet 4). SSE-стриминг.

**Монетизация:** Лимиты AI-запросов (5/день Free, 100/день Pro, безлимит Max).

**Стоимость:** ~$0.01-0.05 за запрос. При 1000 активных юзеров и 100 запросов/день = $30-150/день.

### A2. AI Prompt Score — оценка качества через LLM

> **Статус:** ❌ Не реализовано (проверено 2026-04-11). Есть только текстовый `Analyze` в `usecases/ai/service.go`, числовой score через LLM не сделан.

**Что:** AI оценивает промпт по 6 критериям (специфичность, контекст, структура, ограничения, роль, формат) и даёт числовой score 0-100 + рекомендации по улучшению.

**Монетизация:** Pro-only фича. Keep My Prompts берёт $10/мес за аналог.

**Стоимость:** ~$0.01 за оценку.

**Effort:** Средний (backend endpoint + frontend UI).

### A3. AI Variations — генерация вариантов промпта

> **Статус:** ✅ Реализовано. Backend: `POST /api/ai/variations` → `Service.Variations()` в `usecases/ai/service.go`, system prompt `systemPromptVariationsTemplate` в `usecases/ai/prompts.go`. Frontend: `components/ai/ai-panel.tsx`.

**Что:** "Сгенерируй 3 варианта этого промпта: короткий, детальный, с примерами".

**Монетизация:** Pro-only, расходует AI-лимит.

**Effort:** Средний.

### A4. AI Smart Suggest — рекомендация следующего промпта

> **Статус:** ❌ Не реализовано. Текущий `usecases/search/` — только текстовый поиск, без embeddings и similarity-рекомендаций.

**Что:** После использования промпта AI предлагает "Вам может быть полезен: [похожий промпт]".

**Монетизация:** Pro-only.

**Стоимость:** Высокая (нужны embeddings или анализ каждый раз).

**Effort:** Высокий.

## Часть 2: Фичи без AI (zero LLM-расходов)

Эти фичи дают **сопоставимую ценность** с AI-фичами, но без расходов на API. Могут как заменить AI-фичи, так и дополнить их.

### B1. Prompt Score — rule-based оценка качества (0-100)

> **Статус:** ❌ Не реализовано. Нет функций подсчёта score по regex/heuristics ни в backend, ни в frontend.

**Что:** Оценка промпта по 6 критериям без AI, чисто на regex/heuristics:

| Критерий (0-15 баллов) | Что проверяем | Как |
|------------------------|---------------|-----|
| Роль (role) | "Ты —", "You are", "Act as" | Regex |
| Контекст (context) | Длина > 100 символов, наличие деталей | `len(content)` |
| Формат вывода (output) | "Формат:", "Ответ:", "Output:" | Regex |
| Примеры (examples) | "Пример:", блоки ```, "Example:" | Regex |
| Ограничения (constraints) | "Не делай", "Avoid", ограничения | Regex |
| Переменные (variables) | `{{...}}` template variables | Уже есть парсер |
| **Бонус:** Длина (10 баллов) | 50-500 слов = оптимально | Формула |

**Результат:** Score 0-100 + конкретные советы ("Добавьте формат вывода +15 баллов").

**Монетизация:** Free = базовый score (общий балл). Pro = детальная разбивка + советы.

**Стоимость:** $0. Мгновенно. Работает offline.

**Effort:** Низкий (frontend-only, можно без backend).

**Аналог:** Keep My Prompts берёт $10/мес за AI Prompt Score. Наш rule-based — бесплатный аналог.

### B2. Prompt Linter — real-time подсказки при редактировании

> **Статус:** ❌ Не реализовано. В prompt-editor нет компонента с правилами/warnings.

**Что:** Как ESLint для кода. Пока пользователь пишет промпт, показываем подсказки:

- ⚠️ "Промпт слишком короткий (< 50 символов)"
- ⚠️ "Нет формата вывода — добавьте как AI должен отвечать"
- ⚠️ "Расплывчатый запрос — конкретизируйте задачу"
- 💡 "Совет: добавьте 1-2 примера для лучших результатов"
- ✅ "Есть роль, контекст, формат — отличный промпт!"

**Монетизация:** Free = базовые проверки (3 правила). Pro = все правила + custom rules для команд.

**Стоимость:** $0.

**Effort:** Средний (frontend компонент + набор правил).

### B3. Prompt Formatter — конструктор по шаблону

> **Статус:** ❌ Не реализовано. Редактор промпта — обычное textarea, Wizard/Builder компонента нет.

**Что:** Вместо пустого поля — пошаговый конструктор:

```
📋 Шаблон "Экспертный промпт":
┌─ Роль:         [Ты — senior developer...]
├─ Задача:       [Напиши unit-тесты для...]
├─ Контекст:     [Проект на Go, GORM, Chi...]
├─ Формат:       [Код в блоках ```, пояснения...]
├─ Ограничения:  [Без моков, только integration...]
└─ Примеры:      [Входные данные → ожидаемый результат]
```

Пользователь выбирает шаблон → заполняет секции → получает структурированный промпт.

**Монетизация:** Free = 3 базовых шаблона. Pro = все шаблоны + создание своих.

**Стоимость:** $0.

**Effort:** Средний.

### B4. Snippet Library — переиспользуемые блоки

> **Статус:** ❌ Не реализовано. Нет модели `Snippet`, нет endpoints, нет UI. Starter Templates (готовые промпты в `/welcome`) — это другое.

**Что:** Библиотека готовых блоков для сборки промптов:

```
📦 Роли:        "Ты — senior developer", "Ты — маркетолог", "Ты — tech writer"
📦 Форматы:     "Ответ в JSON", "Markdown таблица", "Пошаговый план"
📦 Ограничения: "Без воды, до 500 слов", "Только на русском"
📦 Тон:         "Деловой", "Дружелюбный", "Технический"
```

Drag & drop или вставка одним кликом. Пользователь создаёт свои сниппеты.

**Монетизация:** Free = встроенные сниппеты. Pro = создание своих + командные сниппеты.

**Стоимость:** $0.

**Effort:** Средний.

### B5. Chrome Extension — вставка промптов прямо в ChatGPT/Claude

> **Статус:** ❌ Не реализовано. В репо нет папок `extension/` / `chrome-extension/`, нет `manifest.json`.

**Что:** Расширение для браузера. Пользователь открывает ChatGPT → иконка ПромтЛаба → выбирает промпт → промпт подставляется в поле ввода. С подстановкой `{{переменных}}`.

**Монетизация:** Pro-only. **Это #1 причина апгрейда у конкурентов** (SpacePrompts, MuseBox, AIPRM).

**Стоимость:** $0 на сервере (расширение работает через существующий API).

**Effort:** Высокий (отдельный проект, но одноразовый).

**Аналоги:** SpacePrompts ($5/мес), MuseBox ($6/мес), AIPRM (freemium) — все монетизируют через extension.

### B6. Экспорт / Импорт

> **Статус:** ❌ Не реализовано. Нет `delivery/http/export/`, нет endpoints `/api/prompts/export`, нет UI для download. В тарифной сетке заявлено как Pro paywall, но код не добавлен.

**Что:** Экспорт промптов в JSON, Markdown, CSV. Импорт из JSON, Notion, текстовых файлов.

**Монетизация:** Уже заявлен в тарифах как paywall ("Без экспорта" в Free). Нужно реализовать.

**Стоимость:** $0.

**Effort:** Низкий.

### B7. Usage Analytics Dashboard

> **Статус:** ⚠️ Частично. **Есть:** модель `PromptUsageLog` в `models/usage_log.go`, миграция `000008_usage_log.up.sql`, счётчик `usage_count` в `Prompt`, страница `pages/history.tsx` (группировка по дням), агрегированный счётчик на `pages/dashboard.tsx`. **Нет:** графиков — `recharts`/`chart.js` не установлены в `frontend/package.json`, нет Top-5 / heatmap / линейных трендов.

**Что:** Графики на основе существующих данных `prompt_usage_log`:

- Топ-5 используемых промптов (bar chart)
- Использований в день/неделю/месяц (line chart)
- Heatmap активности по дням
- Популярные теги (pie chart)
- Тренды: "На этой неделе +40% использований для кода"

**Монетизация:** Pro = персональные графики. Max = командные графики + activity feed.

**Стоимость:** $0 (данные уже собираются).

**Effort:** Средний (frontend графики, backend агрегация).

### B8. Badges / Achievements — расширенная геймификация

> **Статус:** ❌ Не реализовано. Есть только Streaks (`models/streak.go` `UserStreak`) — это отдельная фича, Badges/Achievements как сущности не существуют.

📚 Коллекционер       — Создай 5 коллекций
🏆 Эксперт            — 30 дней подряд
```

**Монетизация:** Free = базовые badges. Pro = все badges + профиль достижений + premium badges.

**Стоимость:** $0.

**Effort:** Низкий-средний (модель + frontend).

### B9. Smart Collections — автоколлекции по правилам

> **Статус:** ❌ Не реализовано. Обычные коллекции есть, но `SmartCollection`/`FilterRule` нет.

**Что:** Автоматические коллекции по правилам:
- "Все промпты с тегом `code` использованные > 5 раз"
- "Промпты без переменных, созданные за последний месяц"

Как Smart Playlists в iTunes — правила, не AI.

**Монетизация:** Pro-only.

**Стоимость:** $0.

**Effort:** Средний.

### B10. Multiple AI Models — выбор модели

> **Статус:** ⚠️ Частично. **Инфраструктура готова:** `AIConfig.Models []ModelConfig` в `infrastructure/config/ai.go`, поле `default_model` в `models/user.go`, компонент `ModelSelector` в `components/ai/ai-panel.tsx` + localStorage. **Не сделано:** `DefaultModels()` в `config/ai.go` возвращает только `anthropic/claude-sonnet-4` — GPT-4o/Gemini/Llama не добавлены в конфиг.

**Что:** Сейчас только Claude Sonnet 4. Разблокировать через OpenRouter: GPT-4o, Gemini, Llama, Mistral.

**Монетизация:** Free = 1 модель (Claude Sonnet 4). Pro = все модели.

**Стоимость:** Та же (OpenRouter), но выбор даёт ценность.

**Effort:** Низкий (backend уже поддерживает, нужен UI для выбора).

## Замена AI-фич на rule-based аналоги

Каждая AI-фича может быть заменена (или дополнена) rule-based аналогом с нулевыми расходами на LLM.

### AI Enhance → Prompt Formatter + Snippet Library

| | AI Enhance (текущее) | Prompt Formatter + Snippets (замена) |
|---|---------------------|--------------------------------------|
| **Что делает** | LLM переписывает промпт целиком | Пользователь собирает промпт из готовых структур и блоков |
| **Стоимость** | $0.01-0.05 за запрос | $0 |
| **Скорость** | 3-10 секунд (SSE stream) | Мгновенно |
| **Контроль** | AI решает что менять | Пользователь контролирует каждую секцию |
| **Offline** | Нет (нужен API) | Да |
| **Обучающий эффект** | Низкий (чёрный ящик) | Высокий (пользователь учится структуре) |
| **Предсказуемость** | Непредсказуемый результат | Детерминированный результат |

**Рекомендация:** Formatter как основа (бесплатно), AI Enhance как premium "полировка" сверху (Pro).

### AI Rewrite → Prompt Linter + Best Practice Hints

| | AI Rewrite (текущее) | Linter + Hints (замена) |
|---|---------------------|------------------------|
| **Что делает** | LLM переписывает в другом стиле/формате | Real-time подсказки что улучшить и как |
| **Стоимость** | $0.01-0.05 | $0 |
| **Момент** | По запросу (post-hoc) | В процессе написания (real-time) |
| **Конкретность** | Общий рерайт | Точечные советы: "Добавьте формат вывода" |
| **Масштаб** | 1 запрос = 1 рерайт | Бесконечно, на каждом промпте |

**Подсказки лinter (rule-based):**
```
Уровень 1 (Free — 3 правила):
  ⚠️  Промпт < 50 символов → "Слишком короткий — AI не получит достаточно контекста"
  ⚠️  Нет слов "формат/ответ/output" → "Укажите в каком формате AI должен ответить"
  ⚠️  Нет роли "ты/you are/act as" → "Задайте роль AI для лучших результатов"

Уровень 2 (Pro — все правила):
  💡  Нет примеров → "Добавьте 1-2 примера — это повышает качество на 40%"
  💡  Нет ограничений → "Укажите чего избегать (длина, стиль, запреты)"
  💡  Есть {{переменные}} → "Отлично! Промпт переиспользуемый"
  💡  > 500 слов → "Промпт очень длинный — попробуйте разбить на chain"
  ⚠️  Множественные задачи → "Один промпт = одна задача. Разделите на несколько"
  ✅  Все критерии → "Промпт отлично структурирован! Score: 92/100"
```

**Рекомендация:** Linter как основа (Free + Pro), AI Rewrite оставить как опцию для тех кому лень самому.

### AI Analyze → Rule-based Prompt Score

| | AI Analyze (текущее) | Prompt Score (замена) |
|---|---------------------|----------------------|
| **Что делает** | LLM анализирует промпт текстом | Числовой score 0-100 по формуле |
| **Стоимость** | $0.03-0.05 | $0 |
| **Воспроизводимость** | Разный ответ каждый раз | Один и тот же score всегда |
| **Геймификация** | Сложно (текстовый ответ) | Легко (число = прогресс-бар) |
| **Сравнение** | Нельзя сравнить два анализа | "Score v1: 45 → v2: 78 (+73%)" |

**Формула расчёта score (детальная):**
```
Score = Роль + Контекст + Формат + Примеры + Ограничения + Переменные + Длина

Роль (0-15):
  0  — нет указания роли
  8  — есть "ты", "you" но без конкретики
  15 — "Ты — senior Go-разработчик с 10+ лет опыта"

Контекст (0-15):
  0  — < 30 символов
  5  — 30-100 символов
  10 — 100-300 символов
  15 — 300+ символов с деталями

Формат вывода (0-15):
  0  — не указан
  8  — упоминается ("ответь списком")
  15 — детальный ("JSON с полями name, score, reason")

Примеры (0-15):
  0  — нет примеров
  8  — 1 пример
  15 — 2+ примера с input/output

Ограничения (0-15):
  0  — нет
  8  — общие ("без воды")
  15 — конкретные ("до 300 слов, только русский, без markdown")

Переменные (0-15):
  0  — нет {{переменных}}
  15 — есть {{переменные}} (промпт переиспользуемый)

Длина (0-10):
  0  — < 20 символов
  3  — 20-50
  7  — 50-200
  10 — 200-1000 (оптимально)
  5  — > 1000 (слишком длинный)

Итого: max 100 баллов
```

**Рекомендация:** Score как основа (отображается на каждой карточке промпта). AI Analyze — для глубокого текстового разбора по кнопке (Pro).
