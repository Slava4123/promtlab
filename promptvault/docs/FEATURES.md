# ПромтЛаб — Каталог фич (104 идеи, ранжировано)

> Полный список возможных улучшений с обоснованием ценности каждой фичи.
> Дата: 2026-04-07
> Порядок: от самых приоритетных к тем, что можно отложить.

---

## Tier-система

Каждая фича отнесена к одному из четырёх tier'ов:

- **🥇 Tier 1 — Critical (1–19)** — must-have, ключевые отличия, retention-критичные, быстрые победы. Делать первыми.
- **🥈 Tier 2 — High Value (20–44)** — сильно улучшают продукт при реалистичном effort. Делать после Tier 1.
- **🥉 Tier 3 — Medium (45–74)** — полезные дополнения, командные фичи, кастомизация. Когда основа стабильна.
- **📦 Tier 4 — Later (75–104)** — сложные, нишевые или зависящие от масштаба. Откладываем на потом.

Метка `[Категория]` указывает на исходную тематическую группу. Метка `(исходно #X)` — старый номер из тематического каталога для traceability.

**Критерии оценки:**
1. **Impact** — повышение retention, activation, решение боли пользователя.
2. **Effort** — сложность реализации (предпочтение быстрым победам).
3. **Соответствие проекту** — текущий стек (Go + React), премиум-позиционирование, российский рынок (Telegram > Slack), единственная модель Claude Sonnet 4.
4. **Уникальность** — отличия от ChatGPT/Notion/конкурентов.

## 🥇 Tier 1 — Critical

### 1. Template variables — `{{placeholder}}` с формой заполнения ✅ 🥇 [Промпты] (исходно #2)
**Зачем:** Промпты часто параметризованы: "Проанализируй код на {{язык}}: {{код}}". Без переменных пользователь вручную заменяет текст каждый раз. С переменными — заполнил форму → получил готовый промпт.
**Получим:** Промпты становятся переиспользуемыми инструментами, а не одноразовыми текстами. Повышает retention.
**Статус:** Реализовано. Парсер `lib/template/parse.ts` (Unicode-aware, single-pass), диалог `components/prompts/use-prompt-dialog.tsx` с live preview, прогресс-счётчиком `(N / M)`, индикаторами незаполненных полей, warning-тостом и persistence значений в localStorage по `prompt.id`. Интегрировано в dashboard, collection-view, prompt-editor.

### 2. Starter templates — готовые промпты ✅ 🥇 [Шаблоны] (исходно #46)
**Зачем:** Новый пользователь открывает пустой dashboard и не знает с чего начать. 30+ готовых промптов по категориям = мгновенная ценность.
**Получим:** Снижение time-to-value с 10 минут до 30 секунд. Критично для retention.
**Статус:** Реализовано. Onboarding wizard на `/welcome` с 4 категориями (Разработчики 8 / Контент и соцсети 8 / Маркетплейсы WB-Ozon 7 / Продукт и аналитика 7 = 30 промптов). Каталог в `embed.FS` (`backend/internal/usecases/starter/catalog.json`), endpoints `GET /api/starter/catalog` и `POST /api/starter/complete` с атомарной транзакцией (создание промптов + маркировка `users.onboarding_completed_at`). ProtectedRoute двунаправленно: новые юзеры → /welcome, прошедшие → /dashboard. Все 30 промптов используют параметры `{{...}}` через парсер из фичи #1.

### 3. Trash/корзина — 30 дней ✅ 🥇 [Безопасность] (исходно #94)
**Зачем:** Случайное удаление = потеря работы. Корзина с 30-дневным retention позволяет восстановить. Как Recycle Bin в Windows.
**Получим:** Защита от ошибок. Спокойствие пользователей.
**Статус:** Реализовано. Soft delete для промптов и коллекций (30 дней retention, автоочистка через PurgeLoop каждый час). Hard delete для тегов (мгновенное удаление + очистка prompt_tags + автоудаление тегов-сирот при обновлении промпта). Страница `/trash` с секциями по типам, кнопки восстановления/окончательного удаления с confirmation dialogs, sidebar badge с реактивным счётчиком. API: `GET/DELETE /api/trash`, `GET /api/trash/count`, `POST /api/trash/{type}/{id}/restore`, `DELETE /api/trash/{type}/{id}`. Кнопка удаления на PromptCard (hover) + в prompt editor. Toast с "Отменить" при удалении с dashboard.

### 4. Закреплённые промпты — pin to top ✅ 🥇 [Организация] (исходно #14)
**Зачем:** 3-5 промптов используются ежедневно. Pin выносит их наверх dashboard, не нужно скроллить или искать.
**Получим:** Быстрый доступ к самому важному. Экономия 10-20 секунд каждый раз.

### 5. Недавние промпты — quick access ✅🥇 [Организация] (исходно #20)
**Зачем:** 70% обращений — к 5 последним промптам. Секция "Недавние" на dashboard даёт мгновенный доступ без поиска.
**Получим:** Снижение friction. Быстрый старт рабочего дня.

### 6. Interactive checklist — шаги для новичка ✅ 🥇 [Онбординг] (исходно #79)
**Зачем:** "1. Создайте первый промпт ✅ 2. Добавьте тег ☐ 3. Попробуйте AI ☐" Чеклист даёт направление и ощущение прогресса. Снижает отток на 15-30%.
**Получим:** Выше activation rate. Больше пользователей доходят до "aha moment".

### 10. PWA — установка на телефон 🥇 [Мобильный] (исходно #84)
**Зачем:** "Добавить на главный экран" превращает сайт в "приложение": отдельная иконка, полноэкранный режим, offline access.
**Получим:** Mobile presence без разработки нативного приложения.

### 11. Telegram-бот 🥇 [Интеграции] (исходно #67)
**Зачем:** В России Telegram — основной мессенджер. Бот позволяет искать и запускать промпты не открывая браузер: `/prompt search code review`.
**Получим:** Доступность в привычной среде. Уникально для российского рынка.

### 12. ✅ MCP-сервер — промпты из AI-клиентов 🥇 [Интеграции] (исходно #65)
**Зачем:** Разработчики хотят обращаться к промптам программно из AI-клиентов (Claude Code, Cursor, Windsurf и др.), CI/CD, скриптов. Промпты как сервис через стандартный протокол MCP.
**Получим:** Developer audience. **Обещано в тарифе Max — реализовано.**
**Статус:** Реализовано. 12 tools (CRUD промптов, коллекций, тегов, поиск, версии), 3 resources, 1 prompt. Аутентификация через API-ключи. Документация: `docs/MCP.md`.

### 14. ✅ Public share links 🥇 [Шаринг] (исходно #50)
**Зачем:** "Посмотри мой промпт" → ссылка без регистрации. Как Google Docs share. Убирает friction для распространения.
**Получим:** Виральность. Каждый shared промпт = потенциальный новый пользователь.
**Статус:** Реализовано. Публичная страница `/s/{token}`, создание/деактивация ссылок, счётчик просмотров, CTA для конверсии. API: `POST/GET/DELETE /api/prompts/{id}/share`, `GET /api/s/{token}`.

### 15. Поисковые подсказки — автокомплит ✅ 🥇 [Поиск] (исходно #24)
**Зачем:** Пользователь начинает вводить "cod" → видит "code review", "coding standards", "code generation". Экономит набор и подсказывает существующие промпты.
**Получим:** Ускорение поиска, обнаружение промптов которые не знал что есть.

### 16. Подсветка match — выделение в результатах ✅ 🥇 [Поиск] (исходно #22)
**Зачем:** При поиске "review" пользователь видит 10 результатов но не понимает где именно совпадение. Подсветка жёлтым показывает match мгновенно.
**Получим:** Быстрее находить нужный результат. Стандарт UX (Google, GitHub, VS Code).

### 17. Streaks — дни подряд ✅ 🥇 [Геймификация] (исходно #74)
**Зачем:** "7 дней подряд используете ПромтЛаб!" Duolingo доказал: streaks повышают retention на 55%. Простейшая механика с мощным эффектом.
**Получим:** Daily active users. Привычка использования.

### 18. In-app feedback — форма ✅ 🥇 [Обратная связь] (исходно #104)
**Зачем:** Кнопка "Отправить отзыв" в приложении. Проще чем искать email или issue tracker. Снижает барьер обратной связи.
**Получим:** Больше обратной связи. Ранее обнаружение проблем.

### 19. Changelog — анонсы обновлений ✅ 🥇 [Обратная связь] (исходно #103)
**Зачем:** "Что нового?" — страница с историей обновлений. Показывает активность разработки. Пользователи видят прогресс.
**Получим:** Retention через новизну. "Продукт живёт".

## 🥈 Tier 2 — High Value


### 21. Semantic search — поиск по смыслу 🥈 [Поиск] (исходно #21)
**Зачем:** Обычный поиск "customer onboarding" не найдёт промпт "адаптация нового клиента". Semantic search понимает синонимы, контекст, намерение.
**Получим:** Находить промпты описывая что нужно, а не вспоминая ключевые слова.


### 23. Prompt linter — проверка качества 🥈 [Промпты] (исходно #4)
**Зачем:** 80% промптов неоптимальны: слишком короткие, без примеров, без формата вывода. Linter подсвечивает проблемы в реальном времени: "Нет формата вывода", "Слишком расплывчато", "Добавьте примеры".
**Получим:** Качество промптов растёт без усилий пользователя. Образовательный эффект — пользователи учатся писать лучше.

### 24. Markdown-заметки к промпту 🥈 [Промпты] (исходно #3)
**Зачем:** Промпт без контекста бесполезен через месяц. Заметки отвечают на "когда использовать", "какие результаты ожидать", "какие ограничения". Критично для команд — новый участник понимает промпт без объяснений.
**Получим:** Повышение переиспользования промптов, снижение времени онбординга в команде.

### 25. Архив — скрыть без удаления 🥈 [Организация] (исходно #16)
**Зачем:** Удалять старые промпты страшно ("а вдруг пригодится"). Архив убирает из основного списка но сохраняет доступ. Как "Архив" в Gmail.
**Получим:** Чистый dashboard без потери данных. Психологический комфорт.

### 26. Иерархические теги — `ai/writing/seo/titles` 🥈 [Организация] (исходно #18)
**Зачем:** Плоские теги ("seo", "writing") не показывают связь. Иерархия `ai → writing → seo → titles` даёт навигацию по дереву. Клик на "writing" показывает все подтеги.
**Получим:** Масштабируемая таксономия. Легко находить промпты в большой базе.

### 27. Расширенные фильтры — по автору, дате, рейтингу 🥈 [Поиск] (исходно #23)
**Зачем:** Базовый поиск недостаточен когда промптов 200+. Фильтры "автор: Вася, модель: Claude, создан: последняя неделя" сужают результат до релевантных.
**Получим:** Профессиональный инструмент поиска. Необходимо для команд.

### 28. Сохранённые фильтры — быстрые пресеты 🥈 [Организация] (исходно #19)
**Зачем:** "Мои избранные промпты по маркетингу за последний месяц" — каждый раз настраивать фильтры утомительно. Сохранённый фильтр = один клик.
**Получим:** Персональные "виды" данных. Экономия времени для power users.

### 29. Вложения — файлы к промпту 🥈 [Промпты] (исходно #11)
**Зачем:** Некоторые промпты требуют контекста в виде скриншотов, CSV, примеров кода. Прикрепление файлов делает промпт самодостаточным пакетом.
**Получим:** Промпт + контекст в одном месте. Не нужно искать файлы отдельно.

### 30. Multi-language — перевод промпта 🥈 [Промпты] (исходно #9)
**Зачем:** Русскоязычные пользователи часто нуждаются в промптах на английском (для кода, документации). Кнопка "Перевести на EN" экономит время и устраняет языковой барьер.
**Получим:** Расширение аудитории, ценность для билингвальных команд.

### 31. Drag & drop сортировка 🥈 [Организация] (исходно #15)
**Зачем:** Пользователи хотят свой порядок: важные наверху, черновики внизу. Без drag & drop порядок определяется датой создания — неудобно.
**Получим:** Персонализация рабочего пространства. Ощущение контроля.

### 32. Top prompts — рейтинг 🥈 [Аналитика] (исходно #60)
**Зачем:** "Какой промпт используется чаще всего?" Рейтинг показывает реальную ценность. Не тот что лучше написан, а тот что реально помогает.
**Получим:** Фокус на ценности. Promotion лучших промптов.

### 33. Usage dashboard — графики 🥈 [Аналитика] (исходно #58)
**Зачем:** "Сколько промптов мы создали за месяц? Сколько использований?" Визуальное представление активности показывает рост или стагнацию.
**Получим:** Данные для принятия решений. Мотивация через прогресс.

### 34. Quick copy — одно нажатие ✅ 🥈 [Мобильный] (исходно #86)
**Зачем:** Главное действие на мобилке — скопировать промпт. Кнопка копирования должна быть максимально доступной: одно нажатие, haptic feedback.
**Получим:** Мобильная эргономика. Быстрый workflow.

### 35. Swipe actions — жесты 🥈 [Мобильный] (исходно #88)
**Зачем:** Свайп влево = удалить, свайп вправо = избранное. Привычные жесты из iOS/Android mail-клиентов.
**Получим:** Нативный mobile feel. Быстрые действия без меню.

### 36. Public roadmap — прозрачный план 🥈 [Обратная связь] (исходно #102)
**Зачем:** "Когда будет API?" — публичный roadmap отвечает на вопросы заранее. Строит доверие и показывает что продукт развивается.
**Получим:** Прозрачность. Доверие пользователей.

### 37. Contextual tooltips — подсказки 🥈 [Онбординг] (исходно #80)
**Зачем:** При первом открытии AI-панели: "Здесь вы можете улучшить промпт с помощью AI. Попробуйте кнопку 'Улучшить'". Исчезает после первого использования.
**Получим:** Self-serve онбординг. Меньше support запросов.

### 38. Best practice hints — подсказки по качеству 🥈 [Онбординг] (исходно #83)
**Зачем:** При создании промпта: "💡 Совет: добавьте формат вывода для лучших результатов". Ненавязчивые hints улучшают качество контента.
**Получим:** Плавное обучение без отрыва от работы.

### 39. Badges — достижения 🥈 [Геймификация] (исходно #75)
**Зачем:** "Первый промпт 🎉", "100 использований 🔥", "Командный игрок 🤝". Визуальные награды мотивируют и показывают прогресс.
**Получим:** Мотивация через recognition. Игровой элемент.

### 40. Readability score — метрика сложности 🥈 [Промпты] (исходно #6)
**Зачем:** Чем проще промпт, тем лучше AI его понимает. Показываем числовой score (как Flesch-Kincaid для текста): "Ваш промпт: 7/10 — хорошо" или "3/10 — слишком сложная структура".
**Получим:** Геймифицированная мотивация улучшать промпты. Объективная метрика качества.

### 41. Prompt formatter — нормализация структуры 🥈 [Промпты] (исходно #5)
**Зачем:** Разные люди пишут промпты по-разному. Formatter приводит к единому стандарту: system role → задача → контекст → формат вывода → примеры. Команды получают консистентность.
**Получим:** Единый стиль промптов в команде. Проще читать, ревьюить, поддерживать.

### 42. Rich text заметки — WYSIWYG редактор 🥈 [Промпты] (исходно #10)
**Зачем:** Plain text заметки ограничены. WYSIWYG позволяет форматирование: заголовки, списки, ссылки, выделение. Удобнее для длинных инструкций.
**Получим:** Более полезные заметки, лучше документированные промпты.

### 43. Webhooks — уведомления о событиях 🥈 [Интеграции] (исходно #66)
**Зачем:** "Когда промпт обновлён → отправить в Slack". Webhooks позволяют реагировать на события без polling.
**Получим:** Интеграция с любыми инструментами. Автоматизация рабочих процессов.

### 44. Import из Notion 🥈 [Интеграции] (исходно #73)
**Зачем:** Многие хранят промпты в Notion. Одноразовый импорт переносит всё в ПромтЛаб без ручного копирования.
**Получим:** Снижение барьера миграции. Конвертация пользователей Notion.

## 🥉 Tier 3 — Medium

### 45. Комментарии к промптам 🥉 [Команды] (исходно #32)
**Зачем:** "Этот промпт не работает с GPT-4o" — без комментариев такой feedback теряется в чатах. Комментарии привязаны к промпту навсегда.
**Получим:** Контекст и знания сохраняются. Команда учится на ошибках друг друга.

### 46. Activity feed — лента действий 🥉 [Команды] (исходно #33)
**Зачем:** "Кто что менял пока меня не было?" Feed показывает: "Анна создала промпт X", "Дмитрий обновил Y", "Саша добавил тег к Z". Как GitHub activity.
**Получим:** Осведомлённость команды. Не нужно спрашивать "что нового".

### 47. @mentions — упоминание коллег 🥉 [Команды] (исходно #34)
**Зачем:** "@Анна, посмотри этот промпт" — уведомление приходит именно ей. Без этого нужно идти в мессенджер и объяснять контекст.
**Получим:** Коммуникация внутри приложения. Меньше переключений между инструментами.

### 48. Approval workflow — статусы 🥉 [Команды] (исходно #35)
**Зачем:** Для критичных промптов (клиентские коммуникации, юридические тексты) нужен процесс: черновик → на ревью → одобрен. Неодобренный промпт нельзя использовать в production.
**Получим:** Governance и compliance. Enterprise-фича для серьёзных команд.

### 49. Kudos/реакции — emoji на промпты 🥉 [Команды] (исходно #38)
**Зачем:** Быстрая обратная связь без написания комментария. 👍 🔥 💡 на промпт = "мне полезно". Показывает какие промпты ценны для команды.
**Получим:** Социальное подтверждение. Мотивация создавать качественные промпты.

### 50. Team analytics — активность команды 🥉 [Аналитика] (исходно #61)
**Зачем:** Менеджер видит: "Анна — 15 промптов, Дмитрий — 3". Не для контроля, а для понимания кто эксперт и где нужна помощь.
**Получим:** Управление командой через данные.

### 51. Cost analytics — расходы на AI 🥉 [Аналитика] (исходно #63)
**Зачем:** "За этот месяц мы потратили $15 на AI. 60% — на команду маркетинга." Прозрачность расходов по командам и пользователям.
**Получим:** Бюджетирование AI-расходов. Необходимо для enterprise.

### 52. Industry templates — по отраслям 🥉 [Шаблоны] (исходно #47)
**Зачем:** Маркетолог и разработчик используют совершенно разные промпты. Отраслевые шаблоны говорят "мы понимаем вашу работу".

### 75. Prompt chains — цепочки промптов 📦 [Промпты] (исходно #12)
**Зачем:** Сложные задачи требуют нескольких шагов: "сначала проанализируй → потом сгенерируй → потом проверь". Chains связывают промпты в последовательность: output одного = input следующего.

### 77. Pull requests — ревью перед merge 📦 [Версии] (исходно #28)

### 78. Inline comments на diff 📦 [Версии] (исходно #29)

### 79. Auto-semantic versioning 📦 [Версии] (исходно #30)

### 80. Changelog — автоматический лог 📦 [Версии] (исходно #31)
