# Оценка готовности к релизу

> Обновлено: 2026-04-14

## Общая оценка: 8.5/10 для MVP платного продукта

**Обновление апрель 2026:** Фаза 13 (подписки и оплата) реализована на бэкенде и фронте.
T-Bank интеграция, модель подписок, quota enforcement, webhook idempotency — всё есть и покрыто
unit-тестами (tbank signature, webhook parsing, extractPlanID, idempotency key).
Остались: коммерческий терминал T-Bank, оферта/152-ФЗ, опциональная фискализация (54-ФЗ).

Приложение технически зрелое — Clean Architecture, 21 юзкейс, 100+ эндпоинтов, CI/CD, error tracking. Но для **платного** продукта есть пробелы.

## Что нужно ДО релиза (блокеры)

### 1. ~~Подписки и оплата~~ — ✅ реализовано (апрель 2026)
Интеграция T-Bank Acquiring API V2 готова:
- `subscription_plans`/`subscriptions`/`payments`/`daily_feature_usage` миграции 000019-000023
- `/api/subscription/{plans,checkout,cancel,downgrade,usage}` + `/api/webhooks/tbank`
- Webhook подпись SHA-256 с исключением Receipt/DATA из расчёта
- Двухфазное сохранение Payment (orphan-защита), conditional UPDATE от race
- plan_id хранится в Payment.ProviderData (не матч по сумме)
- Fail-fast валидация конфига при PAYMENT_ENABLED=true
- Unit-тесты: tbank.VerifyWebhookSignature (8), rawToSigValue (12), extractPlanID (7)

**Остаётся:** получить production-терминал T-Bank (сейчас sandbox), реализовать
автопродление (v2), подключить фискализацию чеков (54-ФЗ) если нужно.

### 2. ~~Квоты и лимиты~~ — ✅ реализовано
`quota.Service` с CheckPromptQuota/CheckCollectionQuota/CheckAIQuota/CheckTeamQuota/
CheckShareLinkQuota/CheckExtensionQuota/CheckMCPQuota. Вызовы из всех handlers,
возвращает 402 + QuotaExceededDialog на фронте. Total (Free) vs Daily (Pro/Max) разделение.

### 3. Публичная оферта и условия возврата
Текущие Terms/Privacy — базовые. Для приёма денег в РФ нужна **публичная оферта** и описание условий возврата. Лучше с юристом.

### 4. Production-терминал T-Bank + IP allowlist
Sandbox работает. Для боевого приёма платежей — одобренный терминал T-Bank и IP allowlist их серверов в webhook middleware.

## Что сильно желательно (не блокер, но влияет на конверсию)

### 5. Онбординг для новых пользователей
Welcome-страница есть, 34 шаблона есть — но нет интерактивного тура. Пользователь заходит и не понимает, что делать первым. Короткий guided tour (3-4 шага) увеличит retention.

### 6. Публичная документация / FAQ
Нет help-центра. Для платного продукта пользователи ожидают хотя бы страницу FAQ и базовую документацию «как начать».

### 7. Email-шаблоны
Сейчас письма — plain text через SMTP. Для платного продукта нужны нормально оформленные HTML-письма (верификация, сброс пароля, уведомление о подписке).
